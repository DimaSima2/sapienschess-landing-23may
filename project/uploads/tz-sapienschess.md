**Адресовано:** дизайнеру и разработчику для совместной работы под кураторством Дмитрия Симакова

## 1. Цель проекта

Сделать лендинг и Telegram-бот регистрации для двухдневного детского шахматного турнира Sapiens Chess (23–24 мая 2026, Москва, СберСити). Лендинг должен:

- Закрывать решение родителя (зарегистрировать ребёнка) в одном визите без звонков и уточнений
- Собирать заявки в воронку клиента (amoCRM + дубль в Google Sheet)
- Поддерживать брендовую стилистику Sapiens Chess
- Работать на российском хостинге без зависимости от иностранных сервисов

**Целевая аудитория:**

- Родители 30–45 лет с детьми 6–17 лет, занимающимися шахматами
- Тренеры шахматных школ Москвы и МО
- Шахматные клубы и кружки (как канал распространения родителям)

**Метрики успеха:**

- ≥ 100 уникальных заявок к 22 мая
- Conversion rate «открыл лендинг → отправил заявку» ≥ 5%
- 0 жалоб на работоспособность формы / бота в день турнира

## 2. Технологический стек

### Frontend

- **Framework:** Next.js 14+ с TypeScript
- **Styling:** Tailwind CSS, дополнительно CSS Modules для уникальных бренд-блоков
- **Сборка:** статический экспорт (`next export`) — деплой как набор HTML/CSS/JS файлов
- **Шрифты:** загрузка через `next/font` — Inter (или близкий бесплатный геометрический sans) + альтернативный font-display: swap
- **Иконки:** Lucide React или Phosphor (open-source)
- **Анимации:** Framer Motion на ключевых блоках (hero, cards), без перегрузки

### Backend

- **Framework:** FastAPI на Python 3.11+
- **Контейнеризация:** Docker (Dockerfile + docker-compose.yml для прод/dev)
- **Валидация:** Pydantic v2
- **Логирование:** structlog или стандартный logging с JSON-форматом
- **Rate-limiting:** SlowAPI или собственный middleware на Redis (если используется)

### Telegram-бот

- **Framework:** aiogram 3.x (асинхронный, современный)
- **Storage:** in-memory FSM (для регистрационного flow), без БД (заявки уходят в amoCRM/Sheet)

### Интеграции

- **amoCRM:** REST API webhook на каждый лид (используем существующую воронку заказчика)
- **Google Sheet:** Google Sheets API v4 через service account
- **TLS:** Let's Encrypt + certbot (автообновление через cron)

### Хостинг

- **VPS заказчика:** российский провайдер (Beget / Selectel / Cloud.ru — на выбор заказчика)
- **OS:** Ubuntu 22.04 LTS или Debian 12
- **Веб-сервер:** nginx (reverse proxy + static)
- **Process manager:** systemd unit для FastAPI и бота (либо supervisord)

## 5. Регистрационная форма

### Поля (10 шт)

| # | Поле | Тип | Обязательное | Валидация |
|---|------|-----|--------------|-----------|
| 1 | ФИО участника | text | Да | min 5 символов, 3 слова через пробел |
| 2 | Дата рождения | date | Да | формат ДД.ММ.ГГГГ или date-picker, не позже 31.12.2007 (для рейтинговых) |
| 3 | Категория | select | Да | 1 из 8 категорий из блока 3 |
| 4 | Телефон для связи | tel | Да | формат +7 (XXX) XXX-XX-XX, маска ввода |
| 5 | Email | email | Да | стандартный email regex |
| 6 | ФИО тренера / Название школы | text | Нет | min 3 символа |
| 7 | ФШР ID | text | Нет (Да для рейтинговых категорий B-G) | формат XXX-NNNN или N-NNNN (без проверки против реестра ФШР) |
| 8 | Промокод | text | Нет | произвольный текст, валидация на стороне Ивана (не на сайте) |
| 9 | Удобный канал связи | radio | Да | TG / Max / WhatsApp / телефон |
| 10 | Согласие на обработку данных | checkbox | Да | ссылка на политику конфиденциальности |

### Поведение формы

- Один экран (не мульти-степ) — родитель видит весь объём сразу
- Inline-валидация на blur (потеря фокуса)
- Submit-кнопка disabled пока обязательные поля не заполнены и согласие не дано
- После отправки — toast «Заявка отправлена» + переход к экрану благодарности
- На ошибке сети — retry-кнопка и сохранение в localStorage чтобы не потерять данные

### Anti-spam

- Honeypot-поле (скрытое поле, заполняется только ботом)
- Rate-limit: не больше 5 заявок с одного IP за час (на backend)
- Cloudflare Turnstile или Yandex SmartCaptcha (опционально, если spam-flow появится)

## 7. Backend API

### Endpoints

#### POST /api/register

**Request body:**

```json
{
  "fio": "Иванов Иван Иванович",
  "birth_date": "2010-05-15",
  "category": "C",
  "phone": "+79991234567",
  "email": "parent@example.com",
  "coach": "Петров П.П. / Школа №42",
  "fshr_id": "MS-12345",
  "promo": "EARLYBIRD",
  "contact_channel": "telegram",
  "consent": true
}
```

**Response (200):**

```json
{
  "status": "ok",
  "lead_id": "amocrm:12345",
  "message": "Заявка принята"
}
```

**Response (400 — validation):**

```json
{
  "status": "error",
  "field_errors": {
    "phone": "Неверный формат телефона"
  }
}
```

**Response (429 — rate limit):**

```json
{
  "status": "error",
  "message": "Слишком много заявок с этого IP, попробуйте через час"
}
```

### Поток обработки заявки

1. Валидация Pydantic-схемы
2. Anti-spam check (honeypot + rate-limit)
3. Параллельно:
   - POST в amoCRM webhook (timeout 5s, 1 retry на network error)
   - Append в Google Sheet через Google Sheets API (timeout 5s, 1 retry)
4. Если оба успешны — return 200
5. Если хотя бы один failed — запись в локальный лог + return 200 (заявка не теряется, оператор разберёт)
6. На полный сбой обоих — return 500 с retry на frontend

### Логирование

- JSON-формат
- Логи в `/var/log/sapienschess/api.log` + ротация через logrotate (10 MB × 7 файлов)
- Без записи PII (email/phone/ФИО) в логи — только lead_id и трейсбэк

## 11. Хостинг и деплой

### Конфигурация сервера

- **VPS:** 2 vCPU, 2 GB RAM минимум, 20 GB SSD (адекватно для трафика до 10к visits/день)
- **OS:** Ubuntu 22.04 LTS
- **Веб-сервер:** nginx с конфигом:
  - Статика лендинга (Next.js export) — root `/var/www/sapienschess/dist`
  - Reverse proxy `/api/*` → FastAPI (`localhost:8000`)
  - Reverse proxy `/bot/*` → бот (если webhook режим)
  - Gzip / Brotli compression
  - HTTP/2 включён
- **TLS:** Let's Encrypt + certbot, автообновление через cron `0 3 * * 0 certbot renew --quiet`
- **Firewall:** ufw — порты 22 (ssh), 80 (http→https redirect), 443 (https)
- **Backup:** ежедневный бэкап логов + конфигов в `/var/backups/sapienschess/` через restic (или rsync)

### Деплой workflow

- **Frontend:** локальный билд `next build && next export` → rsync `dist/` на сервер
- **Backend:** Docker контейнер запускается через systemd unit
- **Бот:** аналогично — Docker + systemd

### Переменные окружения

В `.env` файле на сервере (НЕ в репозитории):

- `AMOCRM_TOKEN`
- `AMOCRM_WEBHOOK_URL`
- `GOOGLE_SHEETS_CREDENTIALS_PATH`
- `GOOGLE_SHEET_ID`
- `TELEGRAM_BOT_TOKEN`
- `RATE_LIMIT_PER_HOUR=5`
- `LOG_LEVEL=INFO`
